# Godrive Knowledge System Synthesis

## Current Thesis

The most useful form of a second brain in the AI era is not a file archive and not a chat log.

It is a maintained Markdown wiki with a clear relationship between evidence, memory, and control.

## Four Layers

- Raw sources hold evidence.
- Markdown notes hold durable memory.
- Agents operate as the control layer.
- Outputs, decisions, and execution are the reason the system matters.

## What Makes This Different From A Basic Notes Repo

- source material remains traceable
- notes are rewritten and synthesized over time
- the system supports active projects instead of passive storage
- agent work is grounded in files that persist after the chat ends
- maintenance is a standing loop, not a one-time cleanup event

## What A Second Brain Is Not

- not a folder full of disconnected files
- not a warehouse of screenshots and exports
- not a permanent chat transcript
- not a dashboard that pretends to be the knowledge itself

## Maintenance Principle

The system should run as a loop:

1. capture or register a source
2. write or update a note
3. promote durable takeaways into areas or synthesis
4. keep links and entrypoints current
5. periodically lint for drift

## Public Starter Principle

This repository should remain a framework layer:

- generic
- safe to publish
- easy to clone
- understandable in minutes

Real personal content belongs in a private instance derived from this starter.
