# Privacy Boundary

This repository is designed to be publishable as a starter template.

Use this file as the boundary check before you commit, publish, or open a pull request from a real personal instance.

## Safe To Keep In The Public Template

- folder structure
- starter notes with generic placeholders
- starter dashboard data with neutral sample metrics
- rules, guides, prompts, and templates
- reusable methods and workflows
- public reference notes that do not expose personal identity or secrets

## Keep Private In Your Own Fork Or Local Copy

- legal names, aliases, phone numbers, emails, addresses
- immigration status, score calculations, visa paths, deadlines
- relationship notes, family stories, health history, therapy notes
- private finances, subscriptions, quotas, cloud credentials, asset registries
- employer-sensitive project notes and private work deliverables
- contact networks, weak ties, recruiting conversations, referral trails
- raw personal documents such as resumes, IDs, medical records, transcripts, screenshots
- OAuth tokens, API keys, `.env` files, secrets, or auth exports

## Framework Layer vs Instance Layer

- **Framework layer** is this starter: structure, guidance, and generic samples.
- **Instance layer** is your real working copy: your data, your sources, your decisions, your private operations.

Keep the framework layer public-safe. Keep the instance layer private by default.

## Recommended Working Pattern

1. Fork or clone this starter.
2. Create your private working copy.
3. Replace sample notes with your own content locally.
4. Register raw-source folders outside the repo where possible.
5. Before publishing, search for private identifiers, scores, paths, and contacts.

## Practical Checks Before Publishing

- Search for your name, aliases, emails, phone numbers, and usernames.
- Search for `partner`, `family`, `medical`, `CRS`, `visa`, `salary`, `subscription`, and similar private markers.
- Verify `dashboard-data/*.json` still contains starter values, not real values.
- Verify `.env`, `.cursor` workspace state, and secret files are ignored.
- Confirm that source notes refer to generic or intentionally public sources only.

## Language And Naming

Technical file names, interface names, and control-layer files should stay in English for compatibility.

Your private note content can be English, Chinese, or mixed. Language choice does not change the privacy rule.
