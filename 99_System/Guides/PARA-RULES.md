# PARA Rules

Use PARA as a practical sorting rule, not as a rigid ideology.

## Buckets

| Bucket | Meaning | Default Rule |
|---|---|---|
| Projects | Active work with a deliverable | Must have a next step |
| Areas | Ongoing responsibility | Review regularly |
| Resources | Reusable reference material | Keep only if future use is plausible |
| Archives | Completed or inactive material | Remove from active views |

## Starter Mapping

- `10_Projects/` is for active execution work.
- `20_Areas/Work/`, `Learning/`, and `Operations/` are the starter areas.
- `30_Resources/` is for reusable methods, references, and supporting notes.
- Archive or delete material that no longer helps the system move.

## Decision Rule

If an item does not have a next step, it probably does not belong in Projects.

If an item is ongoing and repeatedly maintained, it probably belongs in Areas.

If an item is reference material for possible future use, it belongs in Resources.
