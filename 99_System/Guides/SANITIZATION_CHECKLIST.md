# Sanitization Checklist

Before making your version public, review these categories:

## Remove or Replace

- personal names and emails
- company or school names
- client names
- local machine paths
- server addresses
- access tokens and secrets
- screenshots with sensitive information
- raw chat transcripts with private context

## Rewrite as Generic Examples

- project names
- deliverables
- operating procedures
- examples that reveal internal workflows
- file names that expose private context

## Keep Public

- folder structure
- templates
- prompts
- rules
- synthetic examples
- architecture diagrams
- general workflow descriptions
