# Workflow

This file describes the default operating loop for the public starter.

## Core Principle

The system exists to support output, judgment, and execution.

Do not optimize for note count or dashboard ceremony.
Optimize for:

- easier retrieval
- better decisions
- cleaner execution
- durable memory that compounds over time

## Entry Roles

- `README.md` is the public homepage.
- `DASHBOARD.md` is the action-first execution page.
- `INDEX.md` is the fuller internal map.
- `99_System/Guides/LAUNCHPAD.md` explains how the launchpad surfaces fit together.
- starter areas hold ongoing responsibility.
- projects hold active delivery work.
- resources hold reusable references.

## Default Loop

1. Capture into `00_Inbox/`.
2. Register raw sources in `90_Attachments/SOURCE-INDEX.md` when needed.
3. Create or update source notes.
4. Promote durable takeaways into projects, areas, or synthesis notes.
5. Log meaningful system changes.
6. Lint periodically for drift and privacy issues.

## Promotion Rule

Promote a note into the first screen only if it is:

- actively used
- stable enough to be an entrypoint
- important enough to deserve homepage space

The homepage is a control surface, not a complete index.

## Dashboard Relationship

- `dashboard/` is optional.
- the sidebar launchpad is a convenience surface, not the source of truth.
- Markdown remains the source of truth.
- Agents operate on the wiki rather than replacing it.

## Privacy Rule

- Keep the public starter generic.
- Move real personal content into a private instance.
- Keep secrets, tokens, and runtime state out of versioned files.
