# Markdown Style

## Goals

- Write for humans first.
- Keep files easy for AI to parse.
- Favor plain language over clever formatting.

## Conventions

- Use short section headings.
- Prefer bullet lists for checklists and summaries.
- Use links when they help navigation.
- Avoid giant unstructured paragraphs if the note is operational.
- Keep titles descriptive and stable.

## Good Defaults

- one idea or one workflow per file
- obvious file names
- frontmatter only when it adds real value
- minimal tables unless the content truly fits a table
