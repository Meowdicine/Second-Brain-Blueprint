# Glue, Don't Build

This repo follows a simple rule:

> Prefer composing mature tools over rebuilding them.

## Apply the Rule Like This

- Use markdown for durable notes.
- Use Git for history and sharing.
- Use existing editors and cloud tools if they already work.
- Use AI to scaffold and transform knowledge.
- Write custom code only when it clearly improves your workflow.

## Avoid These Traps

- building dashboards before the core note system works
- creating custom databases for simple note routing
- rewriting tools that already solve storage, sync, or editing
- adding automation that makes the system harder to understand

## What The Glue Layer Actually Includes

- folder structure
- prompts
- templates
- naming rules
- review rituals
- lightweight scripts only when justified
