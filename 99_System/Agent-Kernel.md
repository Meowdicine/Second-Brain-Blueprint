# Agent Kernel

This file defines the shared operating kernel for all agent entrypoints in this repository.

## Repo Identity

This repo is a **public-safe starter template** for a Markdown-based second brain that is maintained by agents over time.

It is not a public personal vault.

## Core Model

- Raw sources are evidence.
- Wiki notes are durable memory.
- Agents are the control layer.
- Outputs are the reason the system exists.

## Area Model

Use these starter areas unless the user explicitly chooses another structure:

- `20_Areas/Work`
- `20_Areas/Learning`
- `20_Areas/Operations`

These are starter buckets, not rigid categories.

## Maintenance Loop

Operate through a compounding loop:

1. Capture or register a raw source.
2. Create or update a source note.
3. Update the relevant area, project, or synthesis notes.
4. Record the change in the system log if it is meaningful.
5. Periodically lint for drift, stale claims, missing links, and missing synthesis.

## Public-Safe Defaults

- Do not add personal identity data to the template.
- Do not add private dashboards, scores, or contact registries.
- Keep OAuth, secrets, and runtime state out of versioned files.
- Keep sample content realistic but generic.

## Language Policy

- Technical filenames, paths, interface names, and control-layer files stay in English.
- Content may be written in English, Chinese, or mixed, depending on the user's working style.

## Tooling Policy

- Markdown-first is the default workflow.
- The dashboard is optional.
- Google Drive, Google Tasks, and NotebookLM are documented as reusable defaults, not requirements.

## Quality Bar

- A fresh clone should be understandable within minutes.
- The starter should make sense even without prior context.
- Documentation, control files, and dashboard sample data should describe the same architecture.
